# Напоминания неактивным игрокам (Telegram)

Сообщения уходят **только в личный чат** с ботом (`sendMessage(telegram_id)`), **не в группы**.

## Переменные окружения (Render и т.п.)

| Переменная | По умолчанию | Смысл |
|------------|--------------|--------|
| `INACTIVITY_REMINDER_ENABLED` | включено | `false` — полностью выключить |
| `INACTIVITY_REMINDER_AFTER_HOURS` | `4` | Нет сохранения игры (`/api/save`) дольше N часов → кандидат |
| `INACTIVITY_REMINDER_COOLDOWN_HOURS` | `5` | Не чаще одного напоминания одному человеку раз в N часов |
| `INACTIVITY_REMINDER_BATCH` | `500` | Максимум человек за один прогон |
| `INACTIVITY_REMINDER_CRON_SECRET` | пусто | Секрет для HTTP-cron (см. ниже) |

Текст сообщения задаётся в `bot.js` внутри `runInactivityReminderJob()`.

### Утверждённый текст (как видит игрок в Telegram)

```
⚡ Star to Planet

Давно не было тебя в игре — зайди на пару минут.

Энергия, ежедневные задания и пассивный доход не должны простаивать.

Нажми «Играть» ниже — быстро вернёшься в ритм 🚀
```

Плюс кнопка **«✨ ИГРАТЬ ✨»** (WebApp). Заголовок в коде с тегом `<b>` для жирного «Star to Planet».

## Готовый пример: внешний cron + Render

1. В Render → ваш Web Service → **Environment** добавьте, например:

   `INACTIVITY_REMINDER_CRON_SECRET` = длинная случайная строка (как пароль), например 32+ символа.

2. URL сервиса возьмите из Render (например `https://startoplanet.onrender.com`).

3. В **cron-job.org** (или UptimeRobot, EasyCron и т.д.) создайте задачу:

   - **URL:**  
     `https://ВАШ-ДОМЕН.onrender.com/api/cron/inactivity-reminders?secret=ВАШ_СЕКРЕТ_КАК_В_ENV`
   - **Метод:** GET  
   - **Расписание:** каждые **60 минут** (или 30–120 — как удобно; логика «кому слать» всё равно по БД).

   Альтернатива — заголовок вместо query:

   - URL: `https://ВАШ-ДОМЕН.onrender.com/api/cron/inactivity-reminders`  
   - Заголовок: `x-cron-secret: ВАШ_СЕКРЕТ_КАК_В_ENV`

4. Первый запуск после деплоя: откройте URL в браузере (с секретом) или дождитесь cron. В логах Render появится строка вида `📣 Напоминания неактивным: отправлено …`.

Без `INACTIVITY_REMINDER_CRON_SECRET` endpoint отвечает **503** — это нормально, пока секрет не задан.

На сервере дополнительно раз в час запускается внутренний таймер; на «засыпающем» бесплатном Render он не сработает, пока инстанс спит — поэтому **внешний cron по URL обязателен**, если нужна стабильная рассылка.

## Текст для группы команды (можно вставить в Telegram)

```
Star to Planet — напоминания неактивным (текст зафиксирован)

Что сделали:
• В личку бота уходят напоминания тем, у кого давно не было сохранения прогресса (порог и пауза между пушами настраиваются в env).
• За один прогон до 500 человек; лимит в БД до 2000 через INACTIVITY_REMINDER_BATCH.
• Текст напоминания утверждён (4 блока + кнопка «Играть») — см. раздел «Утверждённый текст» в docs/inactivity-reminders-cron.md; старый черновик «Ты давно не заходил…» снят.
• Инструкция по cron: docs/inactivity-reminders-cron.md

Нужно на проде: задать INACTIVITY_REMINDER_CRON_SECRET и повесить GET на cron-job.org раз в час.
```

Скопируйте блок выше в рабочий чат как есть или подправьте под себя.
